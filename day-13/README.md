# Day 13 - Terminal, Console, Shell & Kernel

## Objective

What was the goal for today? Understand how different system components (terminal, console, shell, kernel) work together to execute user commands.

---

## What I Learned

- Terminal is the interface where users type commands and see output.
- Console is the underlying device/interface (physical or virtual) that handles input/output.
- Shell interprets commands, checks syntax, and sends them to the kernel.
- Kernel is the core of the OS that carries out instructions by interacting with hardware.
- The terminal does not understand commands; it depends on the shell.

---

## What I Built / Practiced

- Practiced running commands in the terminal and observing how input is processed.
- Understood the role each component plays when executing a simple command like ls or pwd.


---

## Challenges Faced

- No deep hands-on interaction with kernel-level operations due to system restrictions.

---

## Key Takeaways

- Clear separation of roles ensures security, stability, and efficiency in the system.
