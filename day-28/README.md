# Day 28 -Advanced Cron Concepts & Monitoring Automation

## Objective

What was the goal for today? To deepen understanding of cron by learning advanced concepts like environment handling and improving automation with real-world logging use cases.

---

## What I Learned

- Cron runs in a limited environment, so variables like $PATH may not behave as expected.
- Always use absolute paths for scripts and commands in cron jobs.
- Cron jobs can fail silently, so proper logging and monitoring are critical.

---

## What I Built / Practiced

- I created a cron job to log system information daily such as memory usage, disk usage and date and user info


---

## Challenges Faced

- I had connection issues while trying to download the report log to my local machine.

---

## Key Takeaways

- Cron is powerful but unforgiving, small mistakes can silently break automation.
- Logging is essential for visibility and debugging.
- Always test scripts before scheduling.
