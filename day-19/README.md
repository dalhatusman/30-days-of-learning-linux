# Day 19 - Introduction to Bash Scripting

## Objective

What was the goal for today? To have a general overview of bash scripting.

---

## What I Learned

- Similar to shell scripting covered few days ago, bash script is a file that contains a sequence of linux commands that are executed together. Bash Script is a specific shell script that executes on a bash interpreter
- Bash Script has two types of variables: Global variable and local variable depending on where it is needed within the script.
- Decision making allows bash scripts to perform different blocks of code based on whether a condition has been met. This makes the script function dynamically.

---

## What I Built / Practiced

- I built a bash script that accepts user input, stores it in a variable called age, and loops through an if-else block to check what the user's age is and print the message as follows::
  - <18: User less than working group
  - >18 and <60: User is among working group 
  - 60: User is a senior

---

## Challenges Faced

- The script was not giving the correct result due to spacing error in the if-else block. 

---

## Key Takeaways

- Bash scripting is about automation, logic, and control
- Once you combine variables with conditions and input/output, you can build real system tools
- Small syntax mistakes can break scripts, so attention to detail is critical
