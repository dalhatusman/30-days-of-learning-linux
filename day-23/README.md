# Day 23 - Difference between sh and bash

## Objective

What was the goal for today? To understand the differences between sh and bash, and know when to use each in shell scripting.

---

## What I Learned

- A shell is an interface between the user and the operating system.
- sh (Bourne Shell) is the original shell and is POSIX-compliant and highly portable.
- bash (Bourne Again Shell) is an enhanced version of sh with more features.
- Scripts written in sh usually run in bash, but not always the other way round.
- Shebang determines the interpreter


---

## What I Built / Practiced

- I wrote simple scripts using both #!/bin/sh and #!/bin/bash.

---

## Challenges Faced

- Understanding why some Bash features don’t work in sh

---

## Key Takeaways

- Use sh when you need maximum portability across systems.
- Use bash when you need advanced scripting features.
- bash is more user-friendly and powerful for real-world automation.
- Always choose the right shell depending on your script requirements.
