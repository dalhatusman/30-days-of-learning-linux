# Day 04 - User Management and File Permission

## Objective

What was the goal for today?

To learn and practice linux user, group and file management
---

## What I Learned

-Linux is a controlled environment that grants privileges to users and groups to have access to the system and its resources. Some actions require administrative (root) privileges
- Concept and commands for granting/revoking privileges to users and groups
- The different types of users and actions they can carry on each file and directory. 

---

## What I Built / Practiced

- Practiced commands to display id, group, name of the user.
- Executed linux commands for creating and switching a user
- Created a file and executed commands to change read/write permissions to user/group/others/all on the file 

---

## Challenges Faced

- Practiced writing commands with sudo privileges which returned error because the user(I) does not have sudoer privileges on the file.

---

## Key Takeaways

- Linux allows multiple users to access a system simultaneously but with controlled privileges making it a secure environment.
- Linux offers flexible permission control by allowing easy modification of ownership and rwx access as users or processes evolve.
