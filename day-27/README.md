# Day 27 - Cron Jobs (Scheduling & Automation)

## Objective

What was the goal for today? To understand how to automate Bash scripts using cron and schedule tasks to run at specific times without manual intervention.

---

## What I Learned

- Cron is a time-based job scheduler in Linux used to automate tasks.
- Cron jobs are defined using a 5-field format: minute, hour, day, month, day of week.
- Cron jobs run in the background and do not display output unless redirected.
- Logging is important using >> and 2>&1 to capture output and errors.
 

---

## What I Built / Practiced

- I practiced scheduling a script using crontab -e

---

## Challenges

- Grasping cron expression format looks difficult.

---

## Key Takeaways

- Automation is critical in data engineering because manual processes don’t scale.
- Always test scripts before scheduling them.
- Logging is not optional; it’s essential for debugging cron jobs.
