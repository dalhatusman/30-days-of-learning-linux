# Day 22 - Functions and Reusability in Bash

## Objective

What was the goal for today? To learn how to use functions in Bash to write reusable, modular scripts and avoid code repetition.

---

## What I Learned

- Functions are reusable blocks of code used to avoid repetition (DRY principle).
- Functions must be defined before they are called.
- Parameters are passed using $1, $2, etc., and $@ for all arguments.
  

---

## What I Built / Practiced

- I built a function to check file existence and log results.

## Key Takeaways
