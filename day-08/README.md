# Day 08 - System Monitoring Tools

## Objective

What was the goal for today?
To learn and practice commands used for monitoring system performance and resource usage and detect issues. 
---

## What I Learned

- System monitoring helps track CPU, memory, disk, and network usage.
- It is useful for detecting bottlenecks and troubleshooting issues.

---

## What I Built / Practiced

- Understood how to identify resource-heavy processes
- Monitored system processes in real time
- Explored disk and CPU activity

---

## Challenges Faced

- I still find it difficult to remember what each command does but with frequent practice, it will stick to the head.

---

## Key Takeaways

- top/htop are go-to tools for quick real-time monitoring.
- Specialized tools (iostat, iftop, iotop) help diagnose specific issues.
- System monitoring is critical for performance tuning and troubleshooting.
