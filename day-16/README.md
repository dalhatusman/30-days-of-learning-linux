# Day 16 - Shebang in Shell Scripting

## Objective

What was the goal for today? To understand the purpose and usage of the shebang (#!) in shell scripts.

---

## What I Learned

- The shebang (#!) defines which interpreter should run a script.
- It must always be the first line in a script with no spaces before it.
- Without a shebang, the system may use a default shell, which can cause errors.

---

## What I Built / Practiced

- Created a simple script using #!/bin/bash and executed it.
- Tested script execution with and without a shebang.
 

---

## Challenges Faced

- Understanding when to use bash vs sh for portability vs functionality.

---

## Key Takeaways

- Shebang is what makes a script behave like an executable program.
- It ensures the correct interpreter is always used, avoiding unexpected errors.
