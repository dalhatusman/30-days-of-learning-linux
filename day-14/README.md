# Day 14 - Writing & Executing Shell Scripts

## Objective

What was the goal for today? Learn how to write and execute basic shell scripts for automation.

---

## What I Learned

- A shell script is a file containing a sequence of commands executed by the shell.
- Used for automation, system tasks, and improving efficiency.
- The shebang (#!/bin/bash) specifies the interpreter. In this case bash shell.
- Variables (name) and input (read) allow interaction with users.

---

## What I Built / Practiced

- Created a script that accepts a number as user input using read and used a simple conditional script (even/odd number checker) to determine if the number is even or odd.

---

## Challenges Faced

- Encountered a syntax error due to a missing closing bracket, which prevented the script from running. 

---

## Key Takeaways

- Shell scripting is powerful for automating repetitive tasks.
- Small syntax mistakes (like missing brackets) can break scripts.
